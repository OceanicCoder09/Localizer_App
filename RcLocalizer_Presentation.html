The Business Problem

Market Context

CAD/BIM software localization is broken under generic tools.

Traditional translation engines lack architectural understanding and terminology context, leading to broken user interfaces and system crashes.

⚠️ "Translating design buttons or placeholders incorrectly halts engineering build pipelines."

⏱️

Slow & Costly

Manual C++ Resource translation takes weeks, involving back-and-forth review cycles between localization agencies and engineers.

🔤

Inconsistent Terms

Terms like "Draft", "Viewport", "Extrude", and "Assembly" translate literally rather than using standard Autodesk vocabulary.

🔄

Repeated Efforts

Localization engines translate the same unchanged string tables repeatedly, wasting thousands of dollars in translation and API fees.

💥

Syntax Crashes

Generic translators corrupt placeholder formats (like %s, %d, {0}) or swap escape sequences (\n, \t), leading to software crashes.

Key Features & Core Components

System Capabilities
🔍

Lexical Parsing

Extracts strings strictly inside STRINGTABLE blocks. Discards comments, macros, and includes, mapping exact char index offsets.

🧠

Intelligent Translate

Queries Gemini in optimized batches. Retries translations through fallback models (3.5-flash, 2.5-flash, 1.5-flash) to guarantee uptime.

📁

Secure Settings

Stores API keys outside source control inside secure user AppData environment files (config.env), migrating legacy keys automatically.

⚡

Smart Local TM

Combines translations and quality feedback in a unified local JSON cache, bypassing the Gemini API entirely for unchanged text segments.

🖥️

Interactive Mockups

Renders a virtual desktop CAD layout dynamically loaded from localized .rc files, supporting live overlays during editing.

🛠️

Bottom-Up Compiler

Generates localized resource files from bottom-to-top. Ensures text alterations do not shift index positions of preceding strings.

How It Works

Operational Sequence

01. Select RC file

Select target file from disk. Parser extracts strings to in-memory grid.

02. AI Translation

Batch queries Gemini, utilizing local caches to avoid repeated API bills.

03. QA Validation

Runs placeholder tests locally and pulls linguistic reports from Gemini QA.

04. Live Preview & Save

Preview overlays, rebuilds C++ files bottom-up, keeping spacing coordinates.

Technology Stack & Core Modules

Technical Implementation

Built on Robust Desktop Foundations

Designed with enterprise scalability, combining native Windows C# structures with cognitive AI orchestration.

💻 C# .NET 10 & WPF: Safe, native Windows desktop framework utilizing MVVM structure.

🧠 Google Gemini API: Multi-model fallbacks (3.5-flash, 2.5-flash, 1.5-flash) to guarantee uptime.

📁 Local Caching: JSON-based local translation memory per target culture code.

📄 Resource Engine: Custom C++ Tokenizer Lexer scanning STRINGTABLE segments.

SYSTEM LOGICAL DATA PIPELINE

1. RcParserService: Scans raw files to identify token coordinate ranges.

2. TranslationMemory: Matches local caches before calling network endpoints.

3. GeminiService REST Client: Submits payloads with specialized CAD prompts.

4. ValidationService QA: Matches parameters count locally and evaluates QA scores.

5. RcGeneratorService: Replaces strings bottom-up to keep coordinates intact.

High-Cohesion Architecture Overview

The application separates logic into strict, dedicated layers to ensure code review readiness:

  • Low Complexity: Methods under 10 lines of code, prioritizing event-driven triggers.
  • Data Model Integrity: Models.cs consolidates all definitions under one namespace, avoiding scattered class files.
  • Dynamic Dropdowns: Dialog previews scan folders dynamically, eliminating hardcoded languages lists.
  • Secure Storage Boundaries: Secrets are isolated in AppData config.env files rather than standard App.config, safeguarding API credentials.
